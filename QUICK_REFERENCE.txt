# 🚀 ESP32 UV Sensor - Quick Reference Card

## Hardware Wiring (3 Connections)
```
UV Sensor          ESP32
VCC (3.3V)    →    3V3
GND           →    GND  
Signal (OUT)  →    GPIO 35 (ADC1_CH7)
```

## ESP32 Code Configuration
**File:** `esp32_uv_sensor.ino`

```cpp
// Line 18 - Change WiFi SSID
const char* ssid = "YOUR_SSID";

// Line 19 - Change WiFi Password  
const char* password = "YOUR_PASSWORD";

// Line 22 - Change Server IP (find with: ipconfig)
const char* serverUrl = "http://192.168.1.100";

// Optional - Change data send interval
const int SEND_INTERVAL = 30000;  // Current: 30 seconds
```

## Upload Steps (5 minutes)
1. **Install Arduino IDE** → https://arduino.cc/en/software
2. **Add ESP32 Board** → File > Preferences > Paste:
   ```
   https://dl.espressif.com/dl/package_esp32_index.json
   ```
3. **Install Library** → Sketch > Include Library > Manage Libraries > Search "ArduinoJson" > Install
4. **Select Board** → Tools > Board > ESP32 > ESP32 Dev Module
5. **Select Port** → Tools > Port > COM3 (or your port)
6. **Upload** → Click Upload button ⬆️
7. **Verify** → Tools > Serial Monitor (115200) > Should see "WiFi connected!"

## API Endpoints
```
POST  /api/sensor-data       ← ESP32 sends data here
      Body: { "uv_reading": 45.5 }
      Status 201 = Success

GET   /api/dashboard-data    ← Frontend fetches from here
      Returns: Current, Average, Total, Recent readings
```

|-------|--------|-------|--------|
| Safe | 🟢 Green | 0-20% | Safe for outdoor |

## Serial Monitor Messages
```
✓ SUCCESS should see:
  "WiFi connected! IP: 192.168.1.XXX"
  "HTTP Response Code: 201"
  "✓ Data sent successfully!"

✗ PROBLEMS:
  "Cannot connect to WiFi" → Wrong SSID/password
  "HTTP Response Code: -1" → Cannot reach server
  "HTTP Response Code: 404" → Wrong endpoint path
  "HTTP Response Code: 422" → Validation error
```

## Find Your Server IP
**Windows:**
```powershell
ipconfig
# Look for "IPv4 Address" - Usually 192.168.1.XXX
```

**Linux/Mac:**
```bash
ifconfig
# Look for "inet" address
```

## Troubleshooting Quick Fixes
```
Problem: No "WiFi connected" message
→ Check COM port in Tools > Port

Problem: Can't upload to ESP32
→ Try different USB cable, restart IDE

Problem: No data on dashboard
→ Check http://localhost:8000/api/dashboard-data in browser

Problem: HTTP -1 error in Serial Monitor
→ Ping server from another device: ping 192.168.1.100
→ Verify WiFi network is 2.4GHz not 5GHz

Problem: HTTP 404 error
→ Check API path is exactly: /api/sensor-data

Problem: HTTP 422 error
→ Check JSON format: { "uv_reading": 45.5 }
```
